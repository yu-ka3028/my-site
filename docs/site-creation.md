# 個人サイトを作った話：技術選定からデプロイまで

## はじめに

AI活用を仕事でもはじめ、ClaudeCodeもどんどん活用したいと思い
以前から興味のあった個人サイトを作成してみました。

---

## 技術選定

| 用途 | 技術 |
|------|------|
| フレームワーク | Astro v5 |
| スタイリング | Tailwind CSS v4 |
| リンター / フォーマッター | Biome |
| デプロイ | Vercel |
| リポジトリ | GitHub |

### なぜ Astro？

- ブログやポートフォリオなど**コンテンツ重視のサイト**に向いている
- デフォルトで JavaScript をほぼ出力しない（静的サイト生成）
- Markdown でブログ記事が書ける
- v5 で Content Collections が使いやすくなっていた

### なぜ Biome？

- ESLint + Prettier の代替として注目していた
- 個人管理なので大規模向きである必要もない
- 設定がシンプルで、導入コストが低い
- Rust 製で高速

## 開発フロー

1. `main` ブランチで作業
2. 修正はフィーチャーブランチを切る（例：`fix-article`）
3. PR を作ってマージ
4. `vercel --prod` でデプロイ
